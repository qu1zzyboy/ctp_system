# 机试题目-基于CTP接口的多账户交易服务

上海宽投资产管理有限公司机试题

v20260703

## 概述

基于 CTP 期货交易接口，实现一个多 Client 的 C/S 架构交易服务系统。Server 端管理多个 CTP 账户的连接与交易，Client 端作为交易终端（代表一个交易者或交易程序）向 Server 发送账户登录、交易指令、查询指令与控制指令。旨在考察候选人对交易接口的对接能力、服务端架构设计水平以及多客户端通信的工程实现能力。

## 题目

### 目标

1. 基于 CTP 交易接口开发，深入阅读接口文档并完成 CTP 的 Trade（交易）和 MD（行情）两个接口的对接与使用。
2. 设计并实现一个多 Client 与单 Server 的 C/S 架构系统。Client 数量不限，测试时使用 2-3 个即可。
3. 各 Client 启动后与 Server 建立连接，发送账户登录请求（含账号密码等登录信息），由 Server 端统一执行 CTP 账户的连接与认证。
4. Client 向 Server 发送交易指令（下单、撤单），Server 执行后将交易反馈路由至对应 Client。
5. Client 向 Server 发送查询指令（资金、持仓、委托、成交），Server 从 CTP 获取结果后返回给对应 Client。
6. Client 向 Server 发送控制指令（如账户断开），由 Server 统一执行。
7. 具备基本的日志功能，记录系统运行与操作记录。
8. CTP 接口按 1 个 MD 行情接口 + 多个 Trade 交易接口的模式接入。

### Server

1. **CTP 接口接入**：接入 CTP 交易接口（Trade）和行情接口（MD），完成初始化、登录、回调处理等完整生命周期管理。按 1 个 MD + 多个 Trade 的模式接入。
2. **多账户管理**：支持多个 CTP 账户同时连接及管理，各账户连接状态独立维护。处理重复连接、重复登录等边界情况。
3. **配置管理**：通过配置文件提供账户登录信息（账号、密码、BrokerID、AppID、AuthCode、交易前置地址、行情前置地址等）。
4. **客户端管理**：对接入的多个 Client 进行连接管理，维护 Client 与 CTP 账户的映射关系。
5. **指令路由**：接收 Client 发来的账户登录、交易指令、查询指令、控制指令，路由到对应的 CTP 账户执行，并将结果反馈路由回对应的 Client。
6. **日志功能**：记录系统运行日志，包括连接状态变更、交易指令记录、异常信息等。
7. 无界面要求。

### Client

1. **自动连接**：启动后自动连接 Server。
2. **账户登录**：向 Server 发送账户登录请求（携带账号密码），由 Server 执行 CTP 账户连接操作。
3. **交易指令**：向 Server 发送下单、撤单指令，并接收 Server 返回的交易反馈（委托状态、成交信息等）。
4. **信息查询**：向 Server 发送查询指令（资金、持仓、委托记录、成交记录），接收并展示查询结果。
5. **控制指令**：向 Server 发送控制指令（如账户断开连接等）。
6. **日志功能**：记录客户端的操作日志。
7. 有无界面均可。

### 附加项

**附加项**（以下为体现区分度的进阶要求，实现越多评分越高）：

1. Server 主动推送订单状态变更到对应 Client，无需 Client 轮询。
2. 断线自动重连机制（Server 端 CTP 连接断开后自动恢复，Client 端与 Server 断开后自动重连）。
3. 多 Client 并发交易指令的排队与冲突处理。
4. Client 交易权限管理（如限制某 Client 只能查询不能交易）。
5. Server 端提供各账户交易汇总面板（命令行或简易界面均可）。
